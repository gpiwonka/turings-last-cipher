## 1. What we are building

A text-based cipher/puzzle adventure for the **DEV June Solstice Game Jam**. The player
decrypts a series of messages that claim to be Alan Turing's last message, helped by an
in-game AI assistant whose trustworthiness erodes over four chapters.

**Twist (spoiler — keep out of player-facing text until the finale):** the messages were not
written by Turing. They were generated by a modern AI reconstructing him. The closing question
mirrors the Turing Test: if an AI can reproduce every thought of a person, is it that person?

### Hard facts that shape decisions
- **Deadline: June 21, 2026, 23:59 PDT.** Short runway. Favor shipping over polish.
- **Prize eligibility requires English.** The game UI **and** the submission post must be in
  English (overall + "Best Ode to Alan Turing" + "Best Google AI Usage"). German is allowed
  only for a non-prize completion badge. → **All player-facing text is English.** A German
  localization may come later as a bonus, never at the cost of the English path.
- Must be a **new** creation built during the jam. Reusing our own snippets is fine but must be
  disclosed in the submission post.
- Handle Turing's biography (persecution, conviction, loss) with dignity. It is the emotional
  core, not a gimmick — this is where the Pride/theme relevance is earned.

## 3. Commands (.NET 9)

```bash
dotnet build                          # build the solution
dotnet test                           # run all xUnit tests
dotnet test --filter "FullyQualifiedName~Enigma"   # run one test class/method
# Test classes available as ~filter targets: CipherRoundTripTests, CipherVectorTests,
# SubstitutionKeyTests, CipherOracleTests, EnigmaTests, StoryAndSessionTests
dotnet run --project src/TuringsLastCipher.Api     # run API + WASM (Development serves the client)
dotnet publish src/TuringsLastCipher.Api -c Release -o publish   # single deployable artifact
```

Notes:
- The hosted WASM client is served automatically in **Development**; in **Production** it is
  served from `wwwroot` only after `dotnet publish` (that's the Cloud Run path). Running the
  unpublished app in Production env will 404 the client — expected, not a bug.
- Deploy is a single container to Google Cloud Run:
  `gcloud run deploy --source . --region us-central1 --set-env-vars GEMINI_API_KEY=...`
  The server reads `PORT` (default 8080) and binds `0.0.0.0` — Cloud Run injects `PORT`.
- This stack matters for scoring: it keeps the build robust, and "deployed on Cloud Run,
  calling the Gemini API" is exactly the story the Google AI category wants. Blazor **WASM**
  (not Server) is deliberate — it avoids the SignalR circuit, fragile on Cloud Run scale-to-zero.

## 4. Architecture

Single ASP.NET Core container hosts the Blazor WASM client (`UseBlazorFrameworkFiles()`) and
exposes `/api/*`. The game is **stateless** server-side: the client tracks the current scene id;
the server is the authority for solution checking and never sends the plaintext to the client.

```
src/
  TuringsLastCipher.Core/    # ciphers, oracle, scene models, deterministic game logic.
                             #   Pure & testable: NO I/O, NO LLM calls, NO ASP.NET deps.
  TuringsLastCipher.Api/     # ASP.NET Core Minimal API: hosts WASM + endpoints + Gemini proxy
  TuringsLastCipher.Client/  # Blazor WASM UI (single terminal-themed game page)
tests/
  TuringsLastCipher.Core.Tests/   # xUnit — the cipher oracle must be bulletproof
content/scenes/story.json  # content-driven story graph (designers add scenes without code)
```

Keep `Core` free of framework/network dependencies so puzzle logic stays deterministic and
unit-testable in isolation. JSON is parsed from **strings** in Core (`StoryJson`); file I/O
lives in `Api` (`StoryProvider`, which loads + validates the story at startup and fails fast).

### Ciphers (Core/Ciphers)
Pure static functions, letters A-Z only, preserve case, pass non-letters through unchanged:
**Atbash**, **Caesar**, **Substitution** (26-letter key permutation), **Vigenère** (keyword;
non-letters don't consume the key), **Enigma-lite**, plus a **frequency** helper (a player
tool, not a cipher). `CipherEngine` dispatches a `CipherSpec` (the content-driven cipher
description) and offers `RoundTrips` as a solvability guard.

Enigma-lite uses historical rotor wirings + reflector + plugboard + ring settings, with
simplified **odometer stepping** (rightmost rotor steps each letter, carries left) instead of
the real notch/double-step. Like the real machine it is **reciprocal** (encrypt twice → plain),
because the reflector is a fixed-point-free involution and the return path inverts the forward
path. It is solved in-game via the `/api/enigma` tool: the player dials rotor start positions
and the server decodes the ciphertext; the secret ring settings stay server-side.

### Oracle (Core)
`CipherOracle.Solved` is the **only** authority on correctness — never the LLM. It normalizes
(uppercase, strip everything but letters/digits) so players aren't punished for formatting.

### Scenes & game state (Core/Scenes, Core/Game)
`Scene` (text + optional `Puzzle` + `Choices` + `Chapter` + optional `HintPolicy`), `Puzzle`
(stores the plaintext truth, derives `Ciphertext`, checks answers), `Story` (immutable graph;
`Validate()` catches dangling links and unsolvable puzzles before shipping), `GameSession`
(deterministic navigation: choices only follow declared links; a puzzle advances only when the
oracle accepts). The **unreliable AI** is scripted: `HintPolicy` (`Truthful | Misleading |
Withholding`) is chosen by `DeceptionDirector` from the chapter (trust erodes Ch1→Ch4); the LLM
only renders the text under that policy.

### Api endpoints (all server-side; key never exposed)
`GET /api/start`, `GET /api/scene/{id}` (returns ciphertext + Enigma metadata, never plaintext
or Enigma positions), `POST /api/solve` (oracle check), `POST /api/hint` and `POST /api/dialogue`
(assistant text under the server-decided policy), `POST /api/frequency` (Core tool), `POST
/api/enigma` (decode tool with player-chosen rotor positions).

### Gemini proxy (Api)
`GeminiClient` is best-effort: missing key / network error / bad response all return null, and
`AssistantService` falls back to static per-policy text so the game is **fully playable
offline** (judges may have flaky networks). Prompts include the cipher name, the genuine hint,
and the policy — **never the plaintext** — so a rendered hint cannot leak the answer, and a
misleading hint cannot block solving (the cipher + frequency tool always provide a path).
Puzzle plaintext generation is a design-time/cached step (generate → review → store in
`content/` → encrypt in C#), never a live dependency.

## 5. Non-negotiable invariants (do NOT violate)

1. **Ciphers are deterministic C#; the LLM never performs crypto.** Encryption, decryption, and
   verification live entirely in `Core`. LLMs corrupt character-level crypto.
2. **Truth lives in code, never in the model.** Every puzzle has a known plaintext in content;
   the oracle is a deterministic comparison. Gemini output is never trusted as ground truth.
3. **Gemini generates content/wording only** — flavor plaintext (finalized & stored before
   encryption), hint phrasing, assistant dialogue. Never puzzle state, gates, or correctness.
4. **The "unreliable AI" is scripted game logic.** The server owns the trust/deception state and
   decides truthful/misleading/withholding; Gemini only renders text. A deceptive hint must
   never make a puzzle unsolvable — a deterministic path to the solution always remains.
5. **No real-world date gates block progression.** The solstice "unlock" is narrative framing
   only; never put a `DateTime.Now` check in front of any scene/ending (judges play later).
6. **The API key never reaches the client.** Gemini is called only server-side via the proxy;
   the Blazor client calls our own `/api/*`, never the Gemini API directly.

## 6. Conventions & priorities

- **KISS, no speculative abstraction.** ~150 lines of cipher logic doesn't need factories or
  one-impl interfaces. `Nullable` and `ImplicitUsings` on; treat warnings seriously.
- **All code, comments, identifiers, and player-facing strings in English** (prize requirement).
- Small pure functions in `Core`; side effects / I/O in `Api`/`Client`. Don't add packages
  without reason — the cipher work needs none.
- **Vertical-slice first.** Priority order: Ch1 (classical) + Ch3 (trust erosion) are the
  thematic core and must ship → Ch4 finale → Ch2 (Enigma-lite) is the cuttable buffer. A small
  finished game beats an ambitious unfinished one. (All four are currently in the slice.)

## 8. Anti-patterns (reject these)

- Asking the LLM to encrypt/decrypt or to judge a player's answer. ❌
- Letting Gemini decide whether a hint lies. ❌
- Calling the Gemini API from the Blazor client. ❌
- Sending the puzzle plaintext (or Enigma positions) to the client or to Gemini. ❌
- A real-world date check that blocks the ending. ❌
- Blazor Server with a persistent circuit on Cloud Run scale-to-zero. ❌
- German player-facing text in the prize submission. ❌
- Over-engineered layers around a few hundred lines of cipher logic. ❌
